# Analysis — talk (exemplar, from scripts)

提供されたお手本を分析し、今後のトーク生成における判断基準となる所見をまとめます。形式的な模倣ではなく、「状況がどう議論を規定するか」という因果関係に重点を置いて整理します。

### 1. 盤面状況と議題の対応
盤面の数値的・論理的状況が、その場の「議題」と「トーン」を決定します。
- **占いCO数（1-CO vs 2-CO以上）**: 1-CO時は「その情報の信頼性」が議題となり、トーンは穏やかです。2-CO以上（対立）になった瞬間、議題は「どちらが真か」という真偽判定に切り替わり、テンポが速まり攻撃的なトーンへ変貌します。
- **黒判定の有無**: 黒判定が出た場合、議題は「黒判定者の反論」と「その反論の妥当性」に集中します。判定がない場合は、発言の違和感やメタ的な戦略論（誰を吊るべきか）という抽象的な議論になります。
- **残り人数と役職分布**: 少人数（5人等）では、1人の誤認が即敗北に繋がるため、「確実な根拠」への要求水準が高まります。また、狂人（Possessed）の存在が、あえて中立を装う、あるいは混乱を煽るというノイズとして機能します。
- **襲撃結果**: 誰が死んだかにより、「占い師が生存しているか（情報の継続性）」と「死者が人間だったか（狼の狙い）」が議論の起点となります。

### 2. 議論展開のパターン
- **カウンタークレイム（対抗CO）**: 占い師に黒判定を出された側が、生存戦略として自らも占い師を名乗るパターン。黒判定が出た直後のTurn 1で発生しやすく、議論を「判定の是非」から「役職の奪い合い」へ強制的に移行させます。
- **行動矛盾の追及**: Day 0での振る舞い（例：占い師を保護しようとする姿勢）と、Day 1での主張（例：突然の占いCO）の整合性を問うパターン。状況証拠を積み上げて論理的に追い詰める際に有効です。

### 3. Dayごとのフェーズ感とTurn進行
- **Day 0**: 【導入】自己紹介、進行ルールの合意、軽い印象操作。
- **Day 1**: 【核心】朝のCO、結果共有、反論への質疑、投票先の絞り込み。
- **Day 2以降**: 【検証】襲撃結果による真偽判定の裏付け、前日の議論との整合性確認。
- **Turn進行**: Turn 0〜2で議論を深め、最終Turn（Turn 3等）は投票宣言に特化させます。残り1発話を投票用に温存し、全員が `@名前` で意思表示して締めるリズムを徹底します。

### 4. 役割の分散と位置取り
全員が同じ方向を向かず、以下のように応答を散らして議論の厚みを出します。
- **応答の散らばり**: 「簡潔な賛同」で流れを支持し、「補足」で論理を強め、「別観点」で死角を提示し、「中立観察」で判断を保留します。
- **主流外の位置取り**: 攻められる側は「非を認める（あるいは絶望的な否定）」、第三者は「整理役（状況の要約）」や「保留（慎重な判断）」を担い、二元論に陥らない多角的な盤面を構築します。

### 5. 発話の組み立てと口調
- **リズム**: 「（相手の発言を引用）＋（それに対する自分の判断材料）＋（短い結論）」という構成で、議論の連鎖を作ります。
- **一貫性**: キャラクターごとの口調（丁寧、論理的、直感的など）を維持しつつ、状況に応じて「冷静な分析」から「切迫した反論」へ感情曲線を変化させます。
