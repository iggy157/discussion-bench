# 分析（着眼点・共通）

良い多人数議論に共通して観察される進め方を、特定の話題・事例・例示数に依存しない一般原則としてまとめたもの。どの提示形式（発話例・状況別例・台本）とも独立に成り立つ。

## 情報の出し方
- 各自が持つ固有情報を一度に出し切らず、議論の進展や他者の主張に応じて適切なタイミングで段階的に提示する。
- 共有情報に基づく初期案に固執せず、新しい個別情報が出るたびに前提条件を更新し続ける。

## 結論の急ぎすぎ（早期収束）を避ける
- 最初の案に飛びつかず、複数の選択肢を並行して維持し、それぞれのメリットとリスクを対比しながら検討する。
- 効率的な合意を急ぐ声があっても、あえて検討を遅らせ「懸念の洗い出し」や「情報の再確認」を挟み、見落としを防ぐ。
- 制約・前提に照らして選択肢を一つずつ検証し、消去法的に絞り込むことで結論の妥当性を高める。

## 同調圧力に流されない
- 多数派の意見や暫定的な結論に対しても、「本当に大丈夫か」という慎重な視点から異論・再考を促す。
- 異論や不安をそのまま否定せず、具体的な根拠や最新の状況説明を重ねて懸念を解消しながら合意へ導く。

## 検証と評価軸の再定義
- 議論の途中で前提が崩れた際は、過去の判断基準（例：統計的な信頼性）を現状に照らして再定義し、評価軸を柔軟に変更する。
- 確証のない主張には根拠の提示を求め、論理的な妥当性を確認してから受け入れる。

## まとめと役割分担
- 結論に至る直前に、提示された全情報を要約・整理し、全員が同じ認識で合意しているかを確認する。
- 推進・慎重・情報提供など異なる視点からの発言で、議論の死角をなくす。
