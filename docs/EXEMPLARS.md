# Exemplars (手本) — script / utterance-fewshot / analysis slots

This tree holds the **hand-curated exemplars** that implement the 3×2 (six-condition)
factorial of the INLG study. They are intentionally **empty for now** — drop files in
later. See `INLG_METHODOLOGY.md` (repo root) for the full design.

この階層は INLG 研究の 3×2（6条件）要因計画を実装する**手本**を置く場所です。
現状は意図的に**空**です（後から作成）。設計は `INLG_METHODOLOGY.md` を参照。

## Layout / 構成

```
agent/<pack>/exemplars/<lang>/<kind>/
  pack   : aiwolf | hidden-bench
  lang   : en | jp
  kind   : scripts     -- full discussion transcripts (台本few-shot; conditions ⑤⑥)
           utterances  -- single-utterance examples (発話few-shot; conditions ③④)
           analysis    -- extracted "where to look / how good discussion flows"
                          (分析; conditions ②④⑥)
```

## Non-negotiable controls (so the paper is defensible) / 防御のための統制

1. **Held-out (leakage control).** Exemplars MUST come from games/tasks **disjoint**
   from the evaluation set. For HiddenBench: if you evaluate task ids X, build scripts
   from the other ids only. / 手本は評価インスタンスと**別**のゲーム/タスクから作る。
2. **Topic-independence.** `analysis/` must describe *how a good discussion proceeds /
   where to look* — never the correct option or task-specific facts (that would leak the
   answer). / 分析は「議論の進め方・着目点」に限定し、正解選択肢・固有事実を含めない。
3. **Token-matching.** `utterances/` (③) and `scripts/` (⑤) are token-matched; the
   `+analysis` pair (④,⑥) is token-matched too — measured with the **discussion agent's**
   tokenizer. Record the per-cell token counts. / ③↔⑤、④↔⑥をエージェントの実トークナイザで一致。
4. **Cross-family generation.** Scripts/analysis are generated by **Claude** (fixed);
   the discussion agent is **Gemma**; the judge is **GPT** — three separate families.

## File format / ファイル形式

- `scripts/*.md` — one full transcript per file (same Markdown convention as
  `data/sample_games_md/`). Loaded via the scenario injection layer.
- `utterances/*.md` (or `.txt`) — short single-utterance examples, one per block.
- `analysis/*.md` — the analysis text injected as the exemplar's AIMessage / summary.

These are consumed by the **same** `_feed_scenario_chunk()` injection mechanism used for
werewolf (identical across domains = cross-domain fairness, see methodology §P2). Wire a
condition by pointing its `scenario.sample_dir` at the relevant `scripts/` directory and
toggling `scenario.enabled`.
